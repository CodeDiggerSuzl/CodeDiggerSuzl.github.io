传统测试&表格测试

传统测试

缺点: - 测试数据和测试逻辑混在一起 - 出错信息不明确 - 一旦一个数据出错,测试全部结束

表格驱动测试

优点: - 分离了测试逻辑和数据,在写测试的时候只用思考数据 - 明确了出错的信息 - 可以部分失败 - go 语法更容易实现表格驱动程序

    代码覆盖率和性能测试

    1. 不用写很多 test 例子,只用写最难的例子
    2. 使用func funcName(b *testing.B){} 来进行性能测试

  3. 命令行

    1. 中使用go test . 来进行测试
    2. 使用go test -coverprofile=c.out查看代码覆盖率
    3. 使用go test -bench . 来进行性测试
    4. 使用go tool -cover=html来在浏览器中查看代码覆盖率

性能提升

  1. 使用go test -bench . -cpuprofile cup.out生成 cpuprofile 的文件.(二进文件)
  2. 使用go tool pprof cpu.out 然后使用 web 来使用浏览器查看(使用brew install graphviz来安装模块)
  3. 可以简单的查看代码慢在哪里

http 测试(🤢)

两种方法: 1. 使用假的 Request 和 Response 来进行测试 类似单元测试 2. 通过起一个服务器 测试覆盖面广

文档

在方法的时候加入注释再使用 go doc/godoc 来查看/生成文档 1. 使用go doc或者go doc 文件名 2. go doc -http:port 3. 可以 随意的写文档 4. 实例代码 也会生成文档 ``func ExampleMethodName() {} 注释中加上 Output: 加入 output 代码运行结果相同的话就会生成文档.
