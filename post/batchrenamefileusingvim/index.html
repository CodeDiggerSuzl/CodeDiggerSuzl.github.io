Multiple Rename & Create Files In Linux

multi create files using touch Week{1..4}.md will create 4 files.

see The Linux Command Line:chapter 8: see the world from shell:expand {}

And I also wanted to rename files without using finder and other GUI tools, I watched a video of LuckSmith which using ranger with it’s Bulk renaming feature.

However, I installed ranger in my mac, and haven’t really to use it, the a saw a post in reddit, I saw a comment below. > Fuck off with that neonazi bullshit. > >Also, for others in this sub (particularly so we have no need to give this piece of shit views), vim can easily do this natively (from :h rename-files): > > 1. :r !ls foo > > 2. <C-v>Imv<Space><Esc> > > 3. Make another column with whatever changes you want to each existing filename (can use :s, etc and combine this step with step 2) > > 4. :w !sh > > This reads the directory into the buffer, prepends the mv command, and once you’ve done your renames, executes the contents of your buffer as shell commands.

This comment is using vim.

In vim we can access shell in normal mode, using :! + command you want to execute.

Using :r to to insert a file,we can also read a output of a linux command. e.g. :r !ls will read the all file names in current directory.

the use visual block mode in vim and insert mv at each beginning. When we want to batch rename files, they tend to have certain rules. like have the same prefix.

Then we append the file name we want to be, like we trim the prefix.

then we use :w !sh to write the text in shell.
