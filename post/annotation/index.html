What The Hell Annotation ?

@since 1.5

  • 作用:

    1. 不是程序本身,但是可以对程序作出解释
      1. 注释是给 coder 看的
      2. 注解是给计算机看的
    2. 可以被其他程序,比如编译器读取
    3. 一个注解准确意义上来说，只不过是一种特殊的注释而已，如果没有解析它的代码，它可能连注释都不如。

  • 格式

    格式是@AnnotationName在代码中存在的,还可以添加参数值,比如@SuppressWarnings(value="unchecked")

  • 注解可以使用在package,class,method,filed上,相当于添加了额外的注释信息

  • 可以通过反射完成对元数据的访问

没错，注解的本质就是一个继承了 Annotation 接口的接口。有关这一点，你可以去反编译任意一个注解类，你会得到结果的。

一个注解准确意义上来说，只不过是一种特殊的注释而已，如果没有解析它的代码，它可能连注释都不如。 而解析一个类或者方法的注解往往有两种形式，一种是编译期直接的扫描，一种是运行期反射。

反射的事情我们待会说，而编译器的扫描指的是编译器在对 java 代码编译字节码的过程中会检测到某个类或者方法被一些注解修饰，这时它就会对于这些注解进行某些处理。

典型的就是注解 @Override，一旦编译器检测到某个方法被修饰了 @Override 注解，编译器就会检查当前方法的方法签名是否真正重写了父类的某个方法，也就是比较父类中是否具有一个同样的方法签名。

这一种情况只适用于那些编译器已经熟知的注解类，比如 JDK 内置的几个注解，而你自定义的注解，编译器是不知道你这个注解的作用的，当然也不知道该如何处理，往往只是会根据该注解的作用范围来选择是否编译进字节码文件，仅此而已。

内置注解

  1. @Override

  2. @Deprecated

已过时,不推荐使用

  1. @SuppressWarning

    扼制警告的作用

  2. @SafeVarargs

    参数安全类型注解。它的目的是提醒开发者不要用参数做一些不安全的操作,它的存在会阻止编译器产生 unchecked 这样的警告。它是在 Java 1.7 的版本中加入的。

  3. @FunctionalInterface

    函数式接口注解，这个是 Java 1.8 版本引入的新特性。函数式编程很火，所以 Java 8 也及时添加了这个特性

    元注解 meta-annotation

『元注解』一般用于指定某个注解生命周期以及作用目标等信息。

元注解就是负责注解其他注解, 4+1 5 个,用来提供对其他注解类型进行说明的作用.在java.lang.annotation包下.

  1. @Target

    • ElementType.TYPE：允许被修饰的注解作用在类、接口和枚举上
    • ElementType.FIELD：允许作用在属性字段上
    • ElementType.METHOD：允许作用在方法上
    • ElementType.PARAMETER：允许作用在方法参数上
    • ElementType.CONSTRUCTOR：允许作用在构造器上
    • ElementType.LOCAL_VARIABLE：允许作用在本地局部变量上
    • ElementType.ANNOTATION_TYPE：允许作用在注解上
    • ElementType.PACKAGE：允许作用在包上

  2. @Retention

    表示需要将该注解报保存到什么级别中,描述注解的生命周期(SOURCE<CLASS<RUNTIME),一般都是用 RUNTIME

    • RetentionPolicy.SOURCE：当前注解编译期可见，不会写入 class 文件
    • RetentionPolicy.CLASS：类加载阶段丢弃，会写入 class 文件
    • RetentionPolicy.RUNTIME：永久保存，可以反射获取

  3. @Document

    它的作用是能够将注解中的元素包含到 Javadoc 中去

  4. @Inherited

说明子类可以继承父类中的注解

Inherited 是继承的意思，但是它并不是说注解本身可以继承，而是说如果一个超类被 @Inherited 注解过的注解进行注解的话，那么如果它的子类没有被任何注解应用的话，那么这个子类就继承了超类的注解。 5. @Repeatable

Repeatable 自然是可重复的意思。`@Repeatable` 是 Java 1.8 才加进来的，所以算是一个新的特性。

通常是注解的值可以同时取多个。

自定义注解

  1. 自动继承了java.lang.Annotation接口
  2. 格式 : public @interface anno{ content}
  3. 其中方法实际上是定义了一个参数
  4. 返回值类型就是参数类型,只能是基本类型(Class,String,enum)
  5. 可以通过default来声明默认值
  6. 如果只有一个参数值,一般参数为 value
  7. 注解元素必须有值,定义注解时候,经常使用空字符串,0,作为默认值

注解的属性

注解的属性也叫做成员变量。注解只有成员变量，没有方法。

注解的成员变量在注解的定义中以“无形参的方法”形式来声明，其方法名定义了该成员变量的名字，其返回值定义了该成员变量的类型。

在注解中定义属性时它的类型必须是 8 种基本数据类型外加 类、接口、注解及它们的数组

注解只是定义 使用注解要靠反射来使用
