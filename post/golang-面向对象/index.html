import "fmt"

type Node struct {
	Value       int
	Left, Right *Node
}

/*
 没有构造函数, 使用工厂函数来构建 Node
 在 cpp 中局部变量的地址是错误的 但是在 go 中不会有问题
*/
func CreateNode(Value int) *Node {
	return &Node{Value: Value}
}

/* 为结构定义方法:
1. 写在结构体的外面
2. 有一个接受者
3. 在方法名前加了一个接收者 代表 this
代表 print 是给 前面括号中接受者来用的
4. node 传递的是值不是引用 所有的都是值传递
*/

func (node Node) Print() {
	fmt.Print(node.Value)
}

/*
 这样定义 set 方法不会改变 node 的值因为传递
 func (node Node)setValue(Value int)  {
		node.Value = Value
 }
 应该使用指针来定义方法
 在调用的时候 解析 地址来设置 Value
 nil 可以调用方法
*/
func (node *Node) SetValue(Value int) {
	if node == nil {
		fmt.Println("nil is inside the func pass")
		return
	}
	// 加不加 * 号都要使用调用
	// 也不用使用*node 来调用 Value
	node.Value = Value
}

/* 也可以这样写 */
func Print(node Node) {
	fmt.Println(node.Value)
}

/* 遍历树 中序遍历 */
func (node *Node) Traverse() {
	if node == nil {
		return
	}
	node.Left.Traverse()
	node.Print()
	node.Right.Traverse()
}

	// 不管是指针实例都是用.
	root.left.right = new(treeNode)

值接受者 和指针接收者

  • 改变内容需要使用指针接收者
  • 结构特别大需要用指针接收者
  • 一致性 :如果有指针接受者,最好都是指针接收者
  • 值接收者是 go 语言特有的
  • 值/指着接受者可以接收者值/指针接收者

GOPATH

  • Windows: %USEPROFILE%\go(windows)
  • Linux :~/go(unix,linux)
  • 官方推荐将所有的项目和第三放的库都放在同一个 gopath 中
  • 也可以将每个项目放在不用的 gopath 中

Linux 设置

  1. gopath
  2. path
  3. gobin

引入包就是在 gopath 中的 src 中找的

使用 go get 获取第三方库

  1. 使用 go get …

  2. 使用gopm来获取无法下载的包

go get github.com/gopmgo/gopm

go get -v github.com/gpmgo/gopm

  1. 使用 go build 来进行编译

  2. go install来产生 pkg 和可执行文件 会放到 bin 中 可以执行的文件

  3. go run来直接编译运行

gopath 下的文件

  • src git repository 或者自己的项目
  • pkg install 产生的文件
  • bin 可执行文件
  • 每个目录下都有一个 main 文件
  • 每个目录下就是一个 package 都可以叫 main
  • 但是只能有一个入口 main 方法
  • go install ./… 表示安装所有的文件
