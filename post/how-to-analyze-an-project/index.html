怎么分析开源项目?

观察开源项目

阅读下列文件中的内容:

  • 查看官网或者文档
  • README.md
  • 使用的协议 需要获取的信息:
  • 项目的作用
  • 项目使用的技术架构

重点关注的文件: - sql - 是否有 maven 依赖 pom 或者 gradle 的文件

下载开源项目

观察项目的目录 1. 用了什么技术架构 2. 数据库时候有 3. 本地环境是否满足需要的环境 1. node.js … 2. java maven redis mysql …

尝试运行项目

  1. 前端
  2. 后端
    1. 导入项目
    2. 分析 common 通用模块
    3. 查看配置文件
      1. 端口
      2. 数据库名字
    4. 是否有 api 文件, swagger 描述文件 等

前后端分离的套路

重点: 找到接口的调用关系

  1. 分析前端,打开控制台,点一个接口,分析调用调用条件
  2. 前后端端口调用不一致
    1. 封装了接口请求 ajax axios

How to find good projects?

  1. 分类
  2. star✨
  3. 收藏数
